# AI 辩论聊天室 —— 设计文档

> 日期：2026-06-26
> 状态：设计已验证，待进入实现规划
> 一句话：扔一个话题，主 agent 自动配齐一桌立场鲜明、懂行的角色，让他们在群聊里真吵起来，真人能随时下场，最后求同存异给出结论。

---

## 0. 核心定位（先想清楚什么是核心）

| 要素 | 定位 | 说明 |
|---|---|---|
| 扔话题 → 自动组队 → 真实交锋 | **亮点** | 用户第一眼的魔法 |
| 多方实时辩论调度 | **技术核心** | 必须啃下的硬骨头，决定体验生死 |
| 真人下场且 AI 真搭理你 | **亮点** | 从"看 AI 表演"变成"和 AI 辩论" |
| persona 鲜明、不趋同 | **必做** | 决定"像不像不同的人" |
| find-skill 动态装配 | **长期护城河，非亮点** | 命中才有、后台看不见；第二阶段做 |

**护城河不是"能装配 skill"，是"能让一桌懂行的人，就任何话题，真吵起来，还让你插得上嘴"。**

## 1. 整体架构与技术栈

```
┌─ 前端：网页群聊 UI ───────────────────┐
│  房间 / 消息流(逐字流式) / "正在输入…" /   │
│  @点名 / 真人输入框 / 角色头像侧栏 / 结论卡 │
└──────────── WebSocket ───────────────┘
              ↕ 实时双向
┌─ 后端核心：辩论调度器(自研) ─────────────┐
│  ① 房间状态机：谁有麦、第几轮、是否该停    │
│  ② 意愿度打分(廉价模型并行) → 仲裁放行1人   │
│  ③ 真人插入 = 最高优先级中断              │
└──────────────────────────────────────┘
              ↕ 调用
┌─ 角色 agent 层(Agent SDK) ────────────┐
│  每个角色 = persona文档 + 联网搜索 +      │
│  (find-skill 装配的领域skill, 二期)       │
│  主 agent(组队器):话题→拆立场→实例化角色  │
│  主持人 agent:控场 + 收敛出结论卡          │
└──────────────────────────────────────┘
```

**技术栈**：全 TypeScript。前端 Next.js + 后端 Node，实时用 WebSocket，角色层用 Agent SDK 的 TS 版（`@anthropic-ai/claude-agent-sdk`）。状态先用轻数据库（SQLite/Postgres）。

**底座选型 = 混合**：自研"辩论调度器"管实时发言权/轮次/真人插话（这是 ai-berkshire 和 Agent SDK 都没给的核心），子 agent 的 persona/skill/工具借力 Agent SDK。

**三个角色别混淆：**
- **主 agent / 组队器**：话题→拆立场→生成角色→征询确认→建房
- **角色 agent**：群里的辩手，各执一词
- **主持人 agent**：控场 + 收敛出结论卡
- **调度器**：不是 LLM，是自研状态机/逻辑，管"谁说话"

## 2. 辩论调度器（核心机制）

事件驱动循环：**每来一条新消息，开一轮"举手→仲裁→只让一个人说"。**

```
新消息落地（AI说完 / 真人发言）
  ↓
① 举手打分（廉价小模型，并行）
   每个在场角色输出：{ want, urgency:0-1, relevance:0-1 }
  ↓
② 仲裁选 1 人（纯逻辑，非LLM）
   候选 = 过硬约束者，按 (urgency × relevance) 排序，取 top
   高分接近时加轻随机 → 制造"抢话"真实感
  ↓
③ 只有中选者用大模型生成完整发言（流式吐到房间）
  ↓
④ 回到①
```

**硬约束（防混乱，全在仲裁层）：**
- 禁止同一角色连发（防霸麦）
- 冷却：刚说过的角色 N 轮内权重打折
- 连续 AI 上限 3-4 条 → 强制停，抛"轮到你了，你站哪边？"交还真人
- 每角色每话题发言配额，防刷屏

**真人 = 最高优先级中断：**
- 真人一发言 → 立刻打断当前调度、作废本轮举手
- 真人发言广播给所有角色，重新开一轮打分，相关角色优先（主持人路由给专长最匹配的 1-2 个）
- 中选角色发言必须先引用/回应真人的具体观点，再接续（防自说自话，且"可被说服但要给理由，不谄媚倒戈"）

**成本账**：每轮"打分"×N（廉价小模型、token 极小），"生成"永远只 1 次（大模型）。加角色不会让成本爆炸。

## 3. 组队器：话题 → 一桌角色

```
你扔进话题（+ 可选：几个人 / 指定角色 / 偏对抗还是偏多元）
  ↓
① 主 agent 拆张力：这话题在哪些轴上有分歧？
  ↓
② 生成角色草案（名字+领域+立场倾向+一句话人设）
  ↓
③ 征询你确认：清单给你看 → 增/删/改立场/换人 → 拍板
   （既能全指定，也能全交给它——默认它提案、你微调）
  ↓
④ 实例化每个角色：persona文档 + find-skill匹配(二期) + 联网搜索
  ↓
⑤ 开场盲写：每个角色【互不可见】先独立写"立场卡"
   （一句话观点 + 最强论据），再进群 → 防一上来互相附和
```

**要点：**
- **默认 2-4 个角色**（Character.AI 教训：角色多会糊脸、情感扁平化）
- **平等同侪**，不设权威角色（有权威→其他人谄媚附和→辩论塌缩）
- **persona 文档 = skill 同一物**：私有库里一个"社会学家·宏观"文档既是人设也是领域技能，find-skill 匹配的就是它

**persona 文档结构**（抄 ai-berkshire 的 `dyp-ask.md`）：身份背景 / 核心信念体系 / 语言风格 / 口头禅 / "不知道什么"（知识边界）/ "不要说自己是 AI"。

## 4. 主持人 agent：收敛、求同存异

辩论必须能收敛——讨论的目的是**求同存异**，不是吵不完。

```
① 控场纠偏：跑题拉回；冷场点名；一边倒时主动 cue 反方
② 推进阶段：开场立场卡 → 自由交锋 → 该收了
③ 收尾产出结论卡（结构化）：
   ✅ 达成的共识点
   ⚔️ 核心分歧（谁 vs 谁，争在哪）
   💪 各方最强论据
   ❓ 仍待验证的问题
```

**原则（抄 ai-berkshire）：** "找矛盾才是最有价值的输出"；每个判断附反面论据，不输出"正确的废话"。

**触发**：你点"总结一下"随时出 / 连续 N 轮自动出 / 真人长时间沉默时出。结论卡渲染成**独立卡片**，不混在聊天流里。

## 5. v1 范围（MVP 闭环）

**✅ v1 必做：**
1. 建房：输话题 → 提案 2-4 角色 → 微调 → 盲写立场 → 进群
2. 角色层：手写 persona 模板注入 + 联网搜索
3. 调度器：举手→仲裁→单人发言 + 硬约束 + 真人插队优先
4. 主持人：控场 + 结论卡
5. 前端：实时群聊 UI、逐字流式、@点名、真人输入框

**⏳ 第二阶段（按价值排序）：**
- find-skill 动态装配 + 私有 skill 库（护城河）
- 跨会话记忆 / 角色成长（a16z：留存核心粘性机制）
- 事实核查 agent（抄 `report_audit` 抽检）
- 异构模型（v1 同一底座 + 不同 persona 已足够）
- 多房间 / 分享 / 社交

**数据模型：**
```
Room   { 话题, 角色清单, 状态(组队/辩论/已收敛) }
Role   { persona文档, 配额/冷却状态 }
Message{ 发言者(角色|真人), 内容, 轮次, @指向 }
ConclusionCard { 共识, 分歧, 各方最强论据, 待验证 }
```

**砍的逻辑**：先验证生死命题——"它吵得好不好看、你插话爽不爽"。验证了再上 find-skill 和记忆去拉护城河和留存。

## 6. 关键研究依据（备查）

- **发言调度**：纯自由抢话会同时发言/刷屏/跑题/成本×N（AutoGen GroupChat 踩坑史、Deliberate Lab 因此改"举手系统"）；Inner Thoughts 证明"意愿度打分"在拟人度/轮次自然度上显著更优。结论：意愿度当**信号**而非**权力**，仲裁者每次只放行一人。
- **角色趋同**：同底座模型 + 只换 prompt 会"语气趋同/假性共识"。可靠性排序：不同底座模型 > 每角色独立知识库 > 说话风格示例 > 头像。防塌缩靠"先盲写后辩 + 平等同侪 + 反共识检验"。
- **节奏**：Nielsen 三阈值（0.1s/1s/10s）；"正在输入…"用在首 token 前，之后转流式；每条回复设 token 上限防啰嗦。
- **留存**：a16z——AI 群聊社交"目前没一个真正跑通"，最大杀手是"新奇悬崖"；持久记忆是核心粘性。
- **ai-berkshire 借鉴**：persona=结构化世界观文档；专家差异靠 prompt 模板注入而非动态挂 skill；视角隔离+事后撞矛盾；二阶 agent 循环（生产→评审→定稿）→ 映射主持人收敛；结构化输出模板+准出闸。它缺的（实时多方互辩、真人介入、动态组队）正是本产品的全部价值。
